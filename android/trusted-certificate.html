# 解决 Android 7.0 无法抓包的问题

在使用 Charles 对 Android APP 抓包时，报错 SSLHandshakeException: Received fatal alert: certificate_unknown。原因是我的测试 Android 系统版本太高了。

在 Android 7.0 版本（API Level 24），调整了受信任证书的安全策略，应用将不再信任用户或管理员添加的 CA 以进行安全连接。

解决办法是将 apk 反编译并修改 AndroidManifest.xml 后再重新打包，步骤如下。

# 1. 反编译 APK 文件

apktool d demo.apk
1

# 2. 修改 AndroidManifest.xml

在 application 标签中添加如下标记

<application android:networkSecurityConfig="@xml/network_security_config">
1

之后在 res/xml 目录下创建 network_security_config.xml 文件

<?xml version="1.0" encoding="utf-8"?>
<network-security-config>
    <base-config cleartextTrafficPermitted="true">
        <trust-anchors>
            <certificates src="system" />
            <certificates src="user" />
        </trust-anchors>
    </base-config>
</network-security-config>

# 3. 重新编译

apktool b demo
1

# 4. 重新签名

jarsigner -verbose -keystore ika.keystore demo/dist/demo.apk ika
1

# 5. 归档对齐

zipalign -v 4 demo/dist/demo.apk demo/dist/demo.new.apk
1

执行完以上全部步骤后重新安装 demo.new.apk ，此时便能够正常抓包了。
