        # Objective-C 拷贝

        拷贝是对原对象产生一个新副本的过程。拷贝的目的和意义是：修改原对象不影响副本对象；修改副本对象同样不影响原对象。

        在 OC 中，对于实现了 NSCopying 协议的对象可以使用 copy 方法；对于实现了 NSMutableCopying 协议的对象可以使用 mutableCopy 方法。两者的区别在于副本对象的可变性。

        # 实例





         








        int main(int argc, const char * argv[]) {
    NSString *str = @"hello";
    NSMutableString *mStr = [NSMutableString stringWithString:@"world"];

    NSLog(@"str: %p, copy: %p", str, [str copy]);  // str: 0x100004030, copy: 0x100004030
    NSLog(@"str: %p, copy: %p", str, [str mutableCopy]);  // str: 0x100004030, copy: 0x100546e20

    NSLog(@"mStr: %p, copy: %p", mStr, [mStr copy]);  // mStr: 0x100545460, copy: 0xb0e43ac3326837d9
    NSLog(@"mStr: %p, copy: %p", mStr, [mStr mutableCopy]);  // mStr: 0x100545460, copy: 0x100546a30

    return 0;
}

        上面代码中除了第 5 行副本对象与原对象是同一个外，其余都是一个新的对象。

        为什么副本对象和原对象会一样？因为拷贝核心需要保证的是两个对象之间不会相互影响，而由于 NSString 本身就是不可变的，所以就直接返回了原对象。

        # 手动实现拷贝

        大部分以 NS 开头的类都已实现了 NSCopying、NSMutableCopying 协议。但是对于我们自己定义的类，则需要手动去实现。

        Dog.h

         





        @interface Dog : NSObject<NSCopying, NSMutableCopying>

@property NSString *name;

@end

        Dog.m

        #import <Foundation/Foundation.h>
#include "Dog.h"

@implementation Dog : NSObject

-(id)copyWithZone:(NSZone *)zone
{
    Dog *dog = [[[self class] allocWithZone:zone] init];

    dog.name = _name;

    return dog;
}

-(id)mutableCopyWithZone:(NSZone *)zone
{
    Dog *dog = [[[self class] allocWithZone:zone] init];

    dog.name = _name;

    return dog;
}

@end

        main.m

        int main(int argc, const char * argv[]) {
    Dog *src = [[Dog alloc] init];
    src.name = @"二狗";

    Dog *copy1 = [src copy];
    Dog *copy2 = [src mutableCopy];
    NSLog(@"%@", copy1.name);  // 二狗
    NSLog(@"%@", copy2.name);  // 二狗

    return 0;
}

        # @property copy 属性

        在 OC 中，@property 有一个 copy 属性：

        @property(nonatomic, copy) Dog *dog;
        • 对于对象：防止对象属性被从外部修改，所以需要拷贝一个副本；
        • 对于 Block：为了将 Block 由栈转移到堆，在堆中的 Block 访问外界变量会进行一次 retain，防止变量被销毁，具体参考Block 一节。
