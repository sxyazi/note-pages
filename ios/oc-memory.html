# Objective-C 内存管理

# MRC

在 iOS 5 之后，已经不再推荐手动管理内存，这里为了学习可以在 Xcode 项目配置中查找 Objective-C Automatic Reference Counting 选项设置为 No 以开启 MRC 支持。

为了便于开发与测试，推荐在 Product -> Scheme -> Edit Scheme -> Diagnostics 中勾选 Zombie Objects 以开启对僵尸对象的调试支持。

# 手动内存管理原则

  • 每个 alloc 对应一个 release
  • 每个 retain 对应一个 release

main.m

int main(int argc, const char * argv[]) {
    Person *p = [[Person alloc] init];  // count: 1
    [p retain];                         // count: 2

    [p release];                        // count: 1
    [p release];                        // count: 0，对象释放

    return 0;
}

# 空指针与野指针

  • 空指针：指针指向了 nil
  • 野指针：指针指向了已经被销毁的实例

向一个空指针发送消息不会引发错误，但向一个野指针发送消息会被抛出错误：

objc[31854]: Person object 0x10055be10 overreleased while already deallocating; break on objc_overrelease_during_dealloc_error to debug
1

代码演示：

int main(int argc, const char * argv[]) {
    Person *p1 = nil;
    [p1 release];  // 不会报错

    Person *p2 = [[Person alloc] init];  // count: 1
    [p2 release];                        // count: 0，对象销毁
    [p2 release];                        // 报错了

    return 0;
}

# 配合 getter/setter

main.m

int main(int argc, const char * argv[]) {
    Person *p1 = [[Person alloc] init];
    Person *p2 = [[Person alloc] init];

    Dog *d = [[Dog alloc] init];  // count: 1
    p1.dog = d;                   // count: 2
    [d release];                  // count: 1

    p2.dog = p1.dog;              // count: 2
    p1.dog = nil;                 // count: 1
    p2.dog = nil;                 // count: 0，释放

    return 0;
}

@implementation Person {
    Dog *_dog;

- (Dog *)dog
{
    return _dog;
}

- (void)setDog:(Dog *)dog
{
    if (dog != _dog) {
        [dog retain];
        [_dog release];

        _dog = dog;
    }
}

@end

# 使用 @property 实现

除了通过手动实现 getter/setter 函数实现外，OC 还提供了内置的 @property 指令自动实现这一功能。

  • readonly：只生成 getter
  • readwrite：生成 getter/setter，默认
  • getter：给生成的 getter 方法起一个别名
  • setter：给生成的 setter 方法起一个别名
  • retain：生成带内存管理的 getter/setter 方法
  • assign：只生成普通的 getter/setter 方法，默认

main.m

int main(int argc, const char * argv[]) {
    Dog *d = [[Dog alloc] init];
    Person *p = [[Person alloc] init];

    p.dog = d;
    [d release];

    p.dog = nil;
    return 0;
}

# 数组对象引用

当将一个对象添加到数组中时，会对其进行一次 retain。当将此对象从数组中移除，或当这个数组被销毁时，对象将会被 release 一次。

int main(int argc, const char * argv[]) {
    NSMutableArray *arr = [[NSMutableArray alloc] init];

    Dog *dog = [Dog new];
    NSLog(@"retainCount: %lu", [dog retainCount]);  // retainCount: 1

    [arr addObject:dog];
    NSLog(@"retainCount: %lu", [dog retainCount]);  // retainCount: 2

    [dog release];
    NSLog(@"retainCount: %lu", [dog retainCount]);  // retainCount: 1

    [arr release];  // 此时，Dog 被销毁了

    return 0;
}

# 自动释放池

自动释放池是一种更加方便内存管理的方式，在其中的对象会在退出 @autoreleasepool 上下文前收到一条 release 消息。

@autoreleasepool {
    Person *p = [[[Person alloc] init] autorelease];
}

// 代码执行到这里，p 就被销毁掉了

# ARC

自动内存管理使人们不再需要关心内存何时释放，这些全部由 Xcode 帮忙搞定。这也是现代项目所推荐的方式。

# 实现原理

在 ARC 模式下，当一个对象实例不再被一个强指针引用时，系统就会自动将其释放。

{
    Person *p = [[Person alloc] init];
}

// 代码执行到这里，p 就被系统自动销毁掉了

# 强指针、弱指针

默认情况下，在 OC 中定义的所有指针都是强指针。使用 __weak 可以声明一个弱指针。

Person *p1;           // 强指针
__strong Person *p2;  // 强指针
__weak Person *p3;    // 弱指针

# @property 指令

针对 @property 方法，也提供了另外的两个参数

  • weak：类似 assign，弱指针引用
  • strong：类似 retain，强指针引用
