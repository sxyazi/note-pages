# Objective-C @property 与自动释放

在学习 OC 内存管理过程中，遇到一个很奇怪的问题，记录一下。以下代码运行并测试于 ARC 模式下，MRC 同样适用。

# 样板代码

main.m

int main(int argc, const char * argv[]) {
    Person *p = [Person new];

    p.dog = [Dog new];
    p.dog = nil;

    NSLog(@"进程结束");
    return 0;
}

# 问题复现

当将上面 main.m 文件中的代码作出如下修改时，打印信息 Dog 被销毁了 神奇般地消失了，而这一切仅仅是因为加了一行 NSLog 而已。

int main(int argc, const char * argv[]) {
    Person *p = [Person new];

    p.dog = [Dog new];
    NSLog(@"%@", p.dog);
    p.dog = nil;

    NSLog(@"进程结束");
    return 0;
}

# 问题分析

根据 Person.h 中对 dog 的声明，为 atomic 类型。OC 为了保证其原子性对 getter 与 setter 方法做了大概如下的处理，对 dog retain 并添加了自动释放。

- (Dog *) dog {
    Dog *d = nil;
    @synchronized(self) {
        d = [[_dog retain] autorelease];
    }
    return d;
}

- (void) setDog:(Dog *)dog {
    @synchronized(self) {
        [dog retain];
        [_dog release];
        _dog = dog;
    }
}

而由于我将 main 函数中默认的 @autoreleasepool 标记删掉了，所以才导致没有自动释放的问题。加回去就好了。

int main(int argc, const char * argv[]) {
    @autoreleasepool {
        Person *p = [Person new];

        p.dog = [Dog new];
        NSLog(@"%@", p.dog);
        p.dog = nil;
    }

    NSLog(@"进程结束");

    return 0;
}
