        # Objective-C @property 与自动释放

        在学习 OC 内存管理过程中，遇到一个很奇怪的问题，记录一下。以下代码运行并测试于 ARC 模式下，MRC 同样适用。

        # 样板代码

        main.m

        int main(int argc, const char * argv[]) {
    Person *p = [Person new];

    p.dog = [Dog new];
    p.dog = nil;

    NSLog(@"进程结束");
    return 0;
}

        Dog.m

        @implementation Dog

- (void)dealloc
{
    NSLog(@"Dog 被销毁了");
}

@end

        Person.h

        @class Dog;

@interface Person : NSObject

@property(strong) Dog* dog;

@end

        上面代码执行完毕后会有如下打印：

        2020-10-01 21:10:00.114880+0800 oc[59329:2648135] Dog 被销毁了
2020-10-01 21:10:00.115209+0800 oc[59329:2648135] 进程结束
Program ended with exit code: 0

        # 问题复现

        当将上面 main.m 文件中的代码作出如下修改时，打印信息 Dog 被销毁了 神奇般地消失了，而这一切仅仅是因为加了一行 NSLog 而已。





         






        int main(int argc, const char * argv[]) {
    Person *p = [Person new];

    p.dog = [Dog new];
    NSLog(@"%@", p.dog);
    p.dog = nil;

    NSLog(@"进程结束");
    return 0;
}

        打印内容：

        2020-10-01 21:14:12.221301+0800 oc[59537:2652438] <Dog: 0x10297ce50>
2020-10-01 21:14:12.221646+0800 oc[59537:2652438] 进程结束
Program ended with exit code: 0

        # 问题分析

        根据 Person.h 中对 dog 的声明，为 atomic 类型。OC 为了保证其原子性对 getter 与 setter 方法做了大概如下的处理，对 dog retain 并添加了自动释放。

        - (Dog *) dog {
    Dog *d = nil;
    @synchronized(self) {
        d = [[_dog retain] autorelease];
    }
    return d;
}

- (void) setDog:(Dog *)dog {
    @synchronized(self) {
        [dog retain];
        [_dog release];
        _dog = dog;
    }
}

        而由于我将 main 函数中默认的 @autoreleasepool 标记删掉了，所以才导致没有自动释放的问题。加回去就好了。


         



         

         






        int main(int argc, const char * argv[]) {
    @autoreleasepool {
        Person *p = [Person new];

        p.dog = [Dog new];
        NSLog(@"%@", p.dog);
        p.dog = nil;
    }

    NSLog(@"进程结束");

    return 0;
}

        或者将 dog 设置为 nonatomic 也可以正常释放。





         



        @class Dog;

@interface Person : NSObject

@property(strong, nonatomic) Dog* dog;

@end

        2020-10-01 21:18:23.371382+0800 oc[60210:2679413] <Dog: 0x1005b4860>
2020-10-01 21:18:23.371811+0800 oc[60210:2679413] Dog 被销毁了
2020-10-01 21:18:23.371869+0800 oc[60210:2679413] 进程结束
Program ended with exit code: 0
