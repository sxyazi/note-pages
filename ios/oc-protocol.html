# Objective-C 协议

协议（Protocol）和其它语言中的接口（Interface）类似，用于定义一些方法的声明。

与其它语言不同的是，在 OC 中的协议除了可以声明哪些方法必须实现外，还可以声明哪些方法可选实现，但都不是强制性的，就算不实现也可以编译，只是会有警告。

# 语法格式

下面代码中 PersonProtocol 为协议名称，NSObject 为该协议继承的协议。

# 类型限定

在类型后面添加 <XXProtocol> 可为类型添加协议限定，表明只有实现了这个协议的对象才被接受，但同样这也不是强制性的，如果传递的对象没有实现这个协议，只会给出警告。

main.m

#import <Foundation/Foundation.h>
#include "Dog.h"
#include "WifeCondition.h"

void marry(id<WifeCondition> wifi)
{
    [wifi cook];
    [wifi wash];

    NSLog(@"所以，我和 %@ 结婚", wifi.name);
}

int main(int argc, const char * argv[]) {
    Dog *dog = [Dog new];
    marry(dog);

    return 0;
}

# 选择性实现

使用 @required 和 @optional 可标记方法是否必须实现。如果没有任何标记，默认为 @required。

PersonProtocol.h

#import <Foundation/Foundation.h>

@protocol PersonProtocol <NSObject>

@required  // 下面开始的方法必须实现
- (void)say;

@optional  // 下面开始的方法可选实现
- (void)play;

@end

当调用一个未实现的方法时会导致程序异常。所以对于可选实现的方法在使用时需要检查是否实现。

int main(int argc, const char * argv[]) {
    Person *p = [Person new];
    [p say];
    [p play];  // 由于上面没有实现，程序异常

    return 0;
}

修改为以下代码解决上述问题：

int main(int argc, const char * argv[]) {
    Person *p = [Person new];
    [p say];

    // 检查是否实现 play 方法
    if ([p respondsToSelector:@selector(play)]) {
        [p play];
    } else {
        NSLog(@"没有实现 play");
    }

    return 0;
}
