# 闭包

闭包通常指：一个函数内的函数，引用了该函数外部所定义的变量。闭包的一般形式：

func counter() -> (Int) -> Int {
    var i = 0
    func plus(_ amount: Int) -> Int {
        i += amount
        return i
    }

    return plus
}

let c = counter()
c(1)  // 1
c(2)  // 3
c(3)  // 6

# 闭包原理

现象说明：上面代码中的 i 原本是一局部变量，但是由于闭包函数 plus 引用了该变量，导致退出 counter 函数后该变量没有被及时销毁。

深入到实现细节是：局部变量 i 存放于栈空间，当编译器检测到 plus 函数引用了该变量，在执行到 return plus 时便将该变量从栈空间原样拷贝了一份到堆空间，之后 plus 所操作的都是堆空间中的副本变量。由于该变量在堆空间中，所以才得以被相对持久的保留。

# 关联值

关联值是指在设置枚举属性的同时，可以附加与之相关联的值，这些值是被计入内存占用的。

enum Score {
    case points(Int)
    case grade(Character)
}
var score = Score.points(95)

MemoryLayout.size(ofValue: score)  // 17，实际占用
MemoryLayout.stride(ofValue: score)  // 24，分配内存，必须是 alignment 的倍数
MemoryLayout.alignment(ofValue: score)  // 8，内存对其

上面代码中 size 的结果是 17 。这个结果是 16+1 的和，具体来说：

  • 16：MemoryLayout<Character>.size 的结果是 16，Character 是整个 enum 中的最大占用大小；
  • 1：由于 enum 中出现多个类型，所以需要耗费 1 个字节来区分具体是哪个类型。当 enum 中的类型小于或等于 1 个时，便不必再占用此字节。或当 enum 中的类型个数超过 0x00...0xFF 范围最大的 256 个时，占用空间也会随之变大。

# 继承

  • 只有类支持继承，子类可以重写父类的方法、属性、下标。重写时必须加上 override 关键字；
  • 针对类型方法、下标，被 class 修饰时可被重写，被 static 修饰时不能被重写，点此了解更多；
  • 被声明为 let 的属性不可被重写，子类重写后的属性权限不能小于父类权限；
  • 子类不能将父类属性（存储、计算）重写为 存储属性，但是可以重写为 计算属性；
  • 子类在重写父类属性时，可为该属性添加属性观察器，子类观察器与父类观察器可并存；
  • 被 final 关键字修饰的类不可被继承，被 final 关键字修饰的方法、属性、下标不可被重写；
class Animal {
    func eat() {
        print("Animal eat")
    }

    subscript(index: Int) -> Int {
        index
    }
}

class Dog: Animal {
    override func eat() {
        super.eat()
        print("Dog eat")
    }

    override subscript(index: Int) -> Int {
        super[index] + 1
    }
}

let animal = Animal()
animal[6]     // 6
animal.eat()  // Animal eat


let dog = Dog()
dog[6]     // 7
dog.eat()  // Animal eat、Dog eat

# 属性

  • 存储属性（Stored Property）

    • 类似于成员变量，存储在实例内存中；
    • 类、结构体 可以 定义存储属性；
    • 枚举 不可以 定义存储属性；
    • 在创建类或结构体时，必须初始化所有存储属性；

  • 计算属性（Computed Property）

    • 本质是方法，不占用实例内存空间；
    • 类、枚举、结构体 都可以 定义计算属性；
    • 声明时只能使用 var 而不能使用 let，因为计算属性的值是在变化的；
struct Circle {
    // 存储属性
    var radius: Double

    // 计算属性
    var diameter: Double {
        get { radius * 2 }
        set { radius = newValue / 2 }
    }
}

var circle = Circle(radius: 5)
print(circle.radius)    // 5
print(circle.diameter)  // 10

circle.diameter = 4
print(circle.radius)    // 2
print(circle.diameter)  // 4

print(MemoryLayout<Circle>.stride)  // 8

# 属性观察器（Property Observer）

可以为非 lazy 的 var 存储属性设置属性观察器

  • willSet：将要修改，默认新值参数名为 newValue；
  • didSet：已经修改，默认老值参数名为 oldValue；
  • 在属性声明、初始化器中初始化属性时，不会触发观察器；
struct Person {
    // 在声明时初始化赋值不会触发观察器
    var name: String = "Lucy" {
        willSet {
            print("willSet", newValue)
        }

        didSet {
            print("didSet", oldValue, name)
        }
    }

    init() {
        self.name = "Lily"  // 在初始化函数中赋值不会触发观察器
    }
}

var person = Person()

// willSet Lina
// didSet Lily Lina
person.name = "Lina"

# 延迟存储属性（Lazy Stored Property）

使用 lazy 关键字可以声明一个延迟存储属性，该属性仅在首次使用时才会被初始化。

  • 被 lazy 标记的属性必须是 var 的，而不能是 let；
  • 当结构体中包含一个延迟存储属性时，此结构体实例变量需要被声明为 var，因为延迟属性需要修改内存；
  • 被声明为 let 的属性要求在实例初始化前就必须有值。同时 lazy 是线程不安全的，多个线程同时访问不能保证只初始化一次；
struct Car {
    lazy var image: Image = {
        return Image(url: "https://www.example.com/a.jpg")
    }()
}

var car = Car()  // 结构体必须声明为 var
car.image        // 仅在初次使用时被初始化

# inout 非存储属性实现

  • 对于没有设置计算属性、属性观察器的参数，直接将属性的内存地址传递给 inout 参数；
  • 对于设置了计算属性 或 属性观察器的参数，采用 Copy In Copy Out 的做法。在调用 inout 函数时，先获取参数的原始值（GET），使用原始值创建一个副本变量。然后将此副本变量的内存地址传递给 inout 参数，函数调用完毕后再将副本的值覆盖回该参数（SET）。
struct Shape {
    var width: Int
    var side: Int {
        willSet { print("willSetSide", newValue) }
        didSet { print("didSetSide", oldValue, side) }
    }
    var girth: Int {
        get {
            print("getGirth")
            return width * side
        }
        set {
            print("setGirth", newValue)
            width = newValue / side
        }
    }
}

func setProperty (_ property: inout Int) {
    property = 20
}

var shape = Shape(width: 10, side: 4)

// willSetSide 20
// didSetSide 4 20
setProperty(&shape.side)

// getGirth
// setGirth 20
setProperty(&shape.girth)

# 实例属性与类型属性

  • 实例属性（Instance Property）：只能通过实例访问

    • 存储实例属性（Stored Instance Property）
      • 包含在实例内存中，每个实例都有一份
    • 计算实例属性（Computed Instance Property）

  • 类型属性（Type Property）：只能通过类型访问

    • 存储类型属性（Stored Type Property）
      • 整个生命周期中，全局只占有一份内存
      • 由于没有初始化方法所以必须设置初始值
      • 默认就是 lazy，在初次使用时才被初始化
    • 计算类型属性（Computed Type Property）

通过 static 关键字定义类型属性。针对类还可以使用 class 定义。

  • static 能修饰 类型方法、计算属性、存储属性；
  • class 能修饰 类方法、计算属性，不能修饰 存储属性。简单来说 class 只能修饰方法（计算属性就是 get、set 方法）；
  • static 修饰的类方法不能被重写，class 修饰的类方法可以被重写。在 protocol 中需要使用 static 修饰类型域中的方法。
struct Car {
    static var count = 0

    init() {
        Car.count += 1
    }
}

let c1 = Car()
let c2 = Car()
let c3 = Car()
print(Car.count)  // 3

# 方法

方法可以被定义在枚举、结构体、类中，方法分为实例方法与类型方法。在方法中可以使用 self 关键字，实例方法 self 指向当前实例，类型方法中的 self 指向当前类型。

  • 实例方法（Instance Method）：通过实例对象调用
  • 类型方法（Type Method）：通过类型调用，使用 static 或 class 关键字定义，点此了解更多。
struct Dog {
    let name: String
    static var count = 0

    init(name: String) {
        Dog.count += 1
        self.name = name
    }

    // 实例方法
    func getName() -> String {
        name   // 等价于：self.count
    }

    // 类型方法
    static func getCount() -> Int {
        count  // 等价于：self.count、Dog.count、Dog.self.count
    }
}

let d1 = Dog(name: "Erha")
print(d1.getName())    // Erha
print(Dog.getCount())  // 1

# 值类型变更（Mutating）

由于结构体和枚举是值类型，默认情况下值类型的属性不能被自身的实例方法所修改。添加 mutating 关键字可以允许这种行为。

enum State {
    case low, middle, high
    mutating func next() {
        switch self {
        case .low    : self = .middle
        case .middle : self = .high
        case .high   : self = .low
        }
    }
}

var state = State.middle
state.next()

print(state)  // high

# 初始化器

类、结构体、枚举都可以定义初始化器。类有 2 种初始化器：指定初始化器（designated initializer）、便捷初始化器（convenience initializer）

每个类至少有一个指定初始化器，指定初始化器是类的主要初始化器。默认初始化器总是类的指定初始化器。类偏向于少量指定初始化器，一个类通常只有一个指定初始化器。

// 指定初始化器
init(parameters) {
    statements
}

// 便捷初始化器
convenience init(parameters) {
    statements
}

# 调用规则

  • 指定初始化器必须从它的直系父类调用指定初始化器
  • 便捷初始化器必须从相同的类里调用另一个初始化器
  • 便捷初始化器最终必须调用一个指定初始化器

# 两段式初始化

第一阶段：初始化所有存储属性

  1. 外层调用指定/便捷初始化器
  2. 分配内存给实例，但未初始化
  3. 指定初始化器确保当前类定义的存储属性都初始化
  4. 指定初始化器调用父类的初始化器，不断向上调用，形成初始化器链

第二阶段：设置新的存储属性值

  1. 从顶层初始化器向下，链中的每一个指定初始化器都有机会进一步定制实例
  2. 初始化器现在能够使用 self（访问、修改它的属性，调用它的实例方法等等）
  3. 最终，链中任何便捷初始化器都有机会定制实例以及使用 self
class Person {
	var height: Double
	var weight: Double?

	init(height: Double, weight: Double?) {
		self.height = height
		self.weight = weight
	}

	convenience init(height: Double) {
		self.init(height: height, weight: nil)
	}
}

class Man: Person {
	var sex: String

	init(sex: String, height: Double, weight: Double?) {
        // 2. 分配内存给实例，但未初始化
		// 3. 指定初始化器确保当前类定义的存储属性都初始化
		self.sex = sex

		// 4. 指定初始化器调用父类的初始化器，不断向上调用，形成初始化器链
		// 5. 从顶层初始化器向下，链中的每一个指定初始化器都有机会进一步定制实例
		super.init(height: height, weight: weight)

		// 6. 初始化器现在能够使用 self（访问、修改它的属性，调用它的实例方法等等）
		self.weight = weight
	}

	convenience init(sex: String, height: Double) {
		self.init(sex: sex, height: height, weight: nil)

		// 7. 最终，链中任何便捷初始化器都有机会定制实例以及使用 self
		self.weight = 10
	}
}

// 1. 外层调用指定/便捷初始化器
var man = Man(sex: "Male", height: 175)

# 安全检查

  • 指定初始化器必须先调用父类初始化器才能为继承的属性设置新值；
  • 指定初始化器必须保证在调用父类初始化器之前，完成对自身所有存储属性的初始化；
  • 便捷初始化器必须先调用同类中的其它初始化器，然后再为任意属性设置新值；
  • 初始化器在第一阶段完成之前，不能调用任何实例方法、不能读取任何实例属性，也不能引用 self。直到第一阶段结束，实例才算完全合法。

# 重写

  • 重写父类指定初始化器时必须加 override；
  • 由于父类的便捷初始化器永远不会通过子类直接调用，因此子类无法重写父类的便捷初始化器。所以当子类中存在一个与父类便携初始化器匹配的初始化器，不用加 override。
class Person {
	var height: Double
	var weight: Double?

	init(height: Double, weight: Double?) {
		self.height = height
		self.weight = weight
	}

	convenience init(height: Double) {
		self.init(height: height, weight: nil)
	}
}

class Man: Person {
	var sex: String

	// 重写指定初始化器需要加 override
	override init(height: Double, weight: Double?) {
		self.sex = "Male"

		super.init(height: height, weight: weight)
	}

	// “重写”便携初始化器不需要加 override
	convenience init(height: Double) {
		self.init(height: height, weight: nil)
	}
}

var man = Man(height: 175)

# required

# 可失败初始化器

  • 类、结构体、枚举都可以使用 init? 定义可失败的初始化器；
  • 可以使用 init! 定义隐式解包的可失败初始化器；
  • 不允许同时定义具有相同参数签名的可失败初始化器和非可失败初始化器；
  • 可以使用非可失败初始化器重写可失败初始化器，反过来不行；
  • 可失败初始化器可以调用非可失败初始化器，非可失败初始化器调用可失败初始化器需要进行解包；
  • 如果初始化器调用一个可失败初始化器导致初始化失败，那么整个初始化流程都将失败，且之后的代码停止运行；

# 下标

使用 subscript 关键字可为枚举、结构体、类添加下标功能，下标与计算属性类似，本质就是方法。

  • subscript 可以接受零个或多个参数，并且可为任意类型；
  • subscript 可以没有 setter，但是必须有 getter。当省略 setter 时 可以简写；
  • subscript 的返回类型决定了：getter 的返回类型、setter 的 newValue 类型；
struct Point {
    var x = 0, y = 0

    subscript(index: Character) -> Int {
        get {
            if index == "x" {
                return x
            } else if index == "y" {
                return y
            }

            return 0
        }

        set {
            if index == "x" {
                x = newValue
            } else if index == "y" {
                y = newValue
            }
        }
    }
}

var p = Point()
p["x"] = 10
p["y"] = 20

print(p.x, p.y)        // 10 20
print(p["x"], p["y"])  // 10 20

# guard 语句

guard 语句和 if 语句类似，它是在判断条件为 false 的情况下执行大括号中的代码。与 if 不同的是，在 guard 语句中进行可选值绑定时，绑定的变量、常量可被用于父级作用域。除此之外 guard 必须配合 return break 等条件语句同时使用。

func login(_ info: [String : String]) {
    guard let username = info["username"] else {
        print("用户名为空")
        return
    }

    guard let password = info["password"] else {
        print("密码为空")
        return
    }

    print("用户名：\(username)，密码：\(password)")
}

login(["username": "zhang"])  // 密码为空
login(["username": "zhang", "password": "123456"])  // 用户名：zhang，密码：123456

# 空合并运算符

使用 ?? 运算符可以检查一个变量或常量是否为 nil，如果是，它将会返回后面变量的值，否则返回自身的值。

空合并运算符的返回值类型会根据后面的变量类型来确定。如果后面变量是可选值，那么返回结果也是可选值，否则将返回非可选值。

var name: String? = "Lucy"

name = nil
print(name ?? "Lisa")  // Lisa
print(name ?? Optional("Lisa"))  // Optional("Lisa")

print("Lucy" ?? Optional("Lisa"))  // Optional("Lucy")
