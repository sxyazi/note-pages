        # Rust 所有权

        # 拷贝

        对于简单类型，Rust 采取的是值拷贝的策略。其中包括：

        • 字符类型，char；
        • 数字类型，如 u32，f64 等；
        • 布尔类型，它的值是 true 和 false；
        • 元组，当其所包含类型均为简单类型时，如：(i32, i32)。
        let x = 5;
let y = x;

        将数字 5 绑定到了 x，并且生成一个 x 的拷贝绑定到了 y。

        # 移动

        对于复杂类型（数据不存在栈上，而是存储于堆中），变量的赋值本质是一个「移动」的过程。这一过程被称为「数据所有权的转移」。

        let s1 = String::from("hello");
let s2 = s1;

        在堆中创建了一个 hello 的字符串，并将变量 s1 指向它。s2 = s1 将 s1 的所有权移交给了 s2，此时变量 s1 失效。这一操作类似于其它语言中的 浅拷贝。

        如果想将堆上的数据执行深度拷贝，可以使用一个叫做 clone 的通用函数，此时 s1 的所有权没有发生变化：

        let s1 = String::from("hello");
let s2 = s1.clone();

        对于函数参数而言，所有权也会被转移：

        fn main() {
  let s = String::from("hello");

  foo(s);  // 此时 s 移动到 foo 函数的 s1 参数
  bar(s);  // s 失效了，发生错误
}

fn foo(s1: String) {}
fn bar(s2: String) {}

        同时，函数的返回值也会被转移所有权：

        fn main() {
  let s = foo();  // foo 将返回值 s1 移动给了 s
}

fn foo() -> String {
  let s1 = String::from("hello");

  s1
}

        # 借用

        有些时候，不需要转移数据的所有权，例如下面这个计算字符串长度的函数：

        fn main() {
  let s = String::from("hello");
  let len = str_len(s);

  println!("Length: {}", len); // 5
  println!("String: {}", s);   // 因为所有权被转移而引发错误
}

fn str_len(s: String) -> usize {
  s.len()
}

        对于这类函数，我希望能够计算字符串长度的同时保留 s 变量的所有权不变。这时候就可以使用 Rust 中的数据「借用」机制。

        fn main() {
  let s = String::from("hello");
  let len = str_len(&s);       // 由 s 更改为 &s

  println!("Length: {}", len); // 5
  println!("String: {}", s);   // hello
}

// 参数类型由 String 更改为 &String
fn str_len(s: &String) -> usize {
  s.len()
}

        新增的这 2 个 & 符号是引用的意思，它们允许你使用但不改变数据所有权。

        默认的引用方式是「不可变」引用。以下代码执行将会报错：

        fn main() {
  let s = String::from("hello");

  change(&s);
  println!("{}", s);
}

fn change(s1: &String) {
  s1.push_str("world");
}

        如果想使引用可变，可添加 mut 关键字。这跟变量由默认「不可变变量」转为「可变变量」所一致：

        fn main() {
  let mut s = String::from("hello");

  change(&mut s);
  println!("{}", s);
}

fn change(s1: &mut String) {
  s1.push_str("world");
}

        不过可变引用有一个很大的限制：在特定作用域中的特定数据只能有一个可变引用。以下代码会报错：

        let mut s = String::from("hello");

let r1 = &mut s;
let r2 = &mut s;

        类似的规则也存在于可变与不可变引用中：不能在拥有特定数据的不可变引用的同时拥有其可变引用。原因很简单，不可变引用不希望数据能被其它人修改。以下代码会报错：

        let mut s = String::from("hello");

let r1 = &s;     // 没问题
let r2 = &s;     // 没问题
let r3 = &mut s; // 错误
