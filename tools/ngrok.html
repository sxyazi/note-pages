# Ngrok 内网穿透工具

Ngrok 是一个开源的反向代理软件，通过它可以实现穿透内网的目的，从而使你的本地机器可以被人们在公网上随意访问和传播。

# 服务端

相对于购买现成的 Ngrok 服务，还可以使用自己的服务器搭建一个只属于你自己的私人代理服务。与公共服务相比它会更加稳定和安全。

首先使用 git 把源码克隆下来，由于 ngrok 使用 Go 语言作为开发语言，所以需要同时安装 Golang 语言环境。

apt install -y git golang
git clone https://github.com/inconshreveable/ngrok
1
2

接着准备好一个做了泛解析的域名，域名指向 IP 为你的服务器的公网 IP 地址。然后设置变量 NGROK_DOMAIN 为你的域名。

export NGROK_DOMAIN="ngrok.domain.com"
1

现在进入到刚才克隆下来的源码目录，生成自签名证书并替换掉默认证书。

cd ngrok

openssl genrsa -out rootCA.key 2048
openssl req -x509 -new -nodes -key rootCA.key -subj "/CN=$NGROK_DOMAIN" -days 5000 -out rootCA.pem
openssl genrsa -out device.key 2048
openssl req -new -key device.key -subj "/CN=$NGROK_DOMAIN" -out device.csr
openssl x509 -req -in device.csr -CA rootCA.pem -CAkey rootCA.key -CAcreateserial -out device.crt -days 5000

cp rootCA.pem assets/client/tls/ngrokroot.crt
cp device.crt assets/server/tls/snakeoil.crt
cp device.key assets/server/tls/snakeoil.key

在创建证书时，若提示 Can't load /root/.rnd into RNG 可执行如下代码解决之：

cd /root
openssl rand -writerand .rnd

最后，编译出服务端的二进制文件并运行：

make release-server
./bin/ngrokd -domain="ngrok.domain.com" -httpAddr=":380" -httpsAddr=":3443" -tunnelAddr=":4443" > /dev/null 2>&1 &

# 客户端

完成服务端搭建后就可以开始使用客户端程序了，使用下面代码编译适用于不同操作系统的二进制文件：

GOOS=darwin GOARCH=amd64 make release-client   # macOS
GOOS=windows GOARCH=amd64 make release-client  # Windows
1
2

将编译好的程序下载到本地机器，然后在同目录下创建 ngrok.conf 配置文件

server_addr: ngrok.domain.com:4443
trust_host_root_certs: false

tunnels:
  git:
    remote_port: 6219
    proto:
      tcp: 192.168.1.33:22

  gogs:
    subdomain: gogs
    proto:
      http: 192.168.1.33:3000
